1 Opening

In this document I will be testting various potential issues with creating accessible and convertable to html by way of make4ht. I have spent some time working with feedback from Googles Gemini working through the potential issues. In particular make4ht is not compatable with \DocumentMetadata{options}; Gemini made a number of helpful suggestions on fixing this in particular the use of \ifdefined\HCode. There are also some potential issues with make4ht and biber; it gave me the smaple code that I am using here. [1] To complie the bibliography properly you need to follow the following steps:

  1. lualatex --shell-escape main.tex
  2. biber main
  3. lualatex --shell-escape main.tex
  4. lualatex --shell-escape main.tex
  5. latexmk -C main.tex
  6. make4ht --lua --shell-escape -c myconfig.cfg -B build_files -d html_output main.tex "mathjax,html5"1

Note that in the above description I didn’t use the minted package to type set the code. It seems to have issues with both make4ht and with creating accessible documents. The tagged pdf project reccomends the fancy verbatim package, fancyvrb, I haven’t played with this yet.

In the upcoming sections I will be checking to see which features are compatible with both the LaTeX tagged pdf project and with make4ht. I have had issues with large projects. I suspect this is due to the size, but I hope that by making the project progressively more complicted I can spot issues as they arise.
